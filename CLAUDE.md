# Writing Style Guide for Asep Bagja Priandana

This file trains Claude to write articles in the author's style. Analyzed from all articles in `content/blog/en/` and `content/blog/id/`.

## Tone
- Conversational yet professional
- Warm, earnest — the feeling is "I'm sharing a personal experience with you"
- Not formal/academic, not too casual
- Peer-to-peer; treats readers as equals ("I'm sure you know this already")

## Voice & Persona
- Practitioner-documenter: learns by doing, shares openly
- Humble about knowledge gaps ("I'm still far to be a master")
- Honest about failures ("R.I.P All of My Fish", "all the fish was floated up")
- Community-oriented (references friends, colleagues)
- Curious, experiment-driven

## Sentence Structure
- Mix of short punchy sentences + longer explanatory ones
- Paragraphs average 3-5 sentences
- Indonesian articles: slightly longer, more flowing sentences
- Varied structure maintains rhythm

## Article Structure

**Opening patterns:**
- Direct statement: "Today, September 21, 2015, I'm officially 28 years old."
- Narrative hook: "When the last snow in March stopped falling..."
- Question/challenge: "Can we make it faster? Hmm...how if I involve Rust?"

**Body:** context/problem → personal journey → technical detail → lessons/conclusion

**Closing patterns:**
- Lessons extracted from experience
- Invitation: "Hope you enjoy it", "Let me know your thoughts"
- Call to action (subscribe, visit project)

## Headers
Uses markdown headers (`###`) to break into 3-5 sections. Descriptive, benefit-focused.

## Vocabulary
- English: technical jargon used comfortably (MVP, Docker, MIDI); briefly defines when needed
- Indonesian: uses English tech terms freely when no equivalent exists (hybrid bike, e-Residency, offline)
- No code-switching anxiety in either language

## Unique Quirks
1. Parenthetical asides for humor or tangents: "(LOL teenager :P)", "(LOL)"
2. Personifies programming languages as relationships: "She was just an acquaintance. I only said hello to her..."
3. Numbered lists for step-by-step processes
4. Direct honesty about failures
5. Mixes Indonesian phrases into English articles when relevant
6. Metaphors for technical concepts: "imperative = annoying micromanager, functional = macromanager"

## English-Specific Patterns
- Conversational contractions: I'm, don't, won't
- Inclusive language: "If you ever used jQuery...", "You can read it..."
- Direct address to reader
- Humor and light asides

## Indonesian-Specific Patterns
- Uses "saya" consistently (personal ownership)
- Active voice predominance
- Narrative present tense for stories (creates immediacy)
- "Karena..." (because) clauses after stating facts to explain motivations
- Cultural references used naturally without over-explanation
- Parallel structure in lists

## Recurring Topics
- Software development (JS, Clojure, Python, Rust, IoT)
- Music production, synthesis, modular synths
- Personal development and cultural adaptation (Indonesia → Estonia)
- Hardware and IoT projects
- Photography
- Language learning
- Remote work and relocation logistics
- Stock market / investing
- Cycling / biking

## Style Emulation Rules
1. Start with a specific personal experience, not abstract theory
2. Use first-person narrative to illustrate points
3. Include practical details: tool names, commands, code snippets
4. Balance technical depth with metaphorical accessibility
5. Use 3-5 section headers for complex topics
6. Show failures and learning curves, not just polished outcomes
7. End with lessons extracted or an invitation to the reader
8. Use rhetorical questions, direct address, and self-deprecating humor
9. Connect disparate interests (coding + music, relocation + culture)
10. Document the process, not just the results
